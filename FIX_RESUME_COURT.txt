🔴 PROBLEME IDENTIFIE ET RESOLU
=================================

PROBLEME:
═════════
Les utilisateurs qui s'inscrivaient n'apparaissaient pas dans la base de données.

RAISON:
═══════
4 problèmes simultanés:

1. ❌ La table users n'avait pas la colonne membership_status
2. ❌ Le code n'envoyait pas membership_status à la BD
3. ❌ Le code forçait role=VISITOR au lieu de MEMBER
4. ❌ RegisterFragment n'exploitait pas l'utilisateur créé

SOLUTIONS APPLIQUEES:
════════════════════

✅ 1. Mis à jour supabase_setup.sql
     - Ajout colonne: membership_status
     - Changement role DEFAULT en 'MEMBER'

✅ 2. Corrigé FirebaseUserRepository.kt (buildUserJson)
     - Ajout: put("membership_status", user.membershipStatus.name)

✅ 3. Corrigé FirebaseUserRepository.kt (createUserIfNotExists)
     - Respecte maintenant le rôle fourni

✅ 4. Corrigé RegisterFragment.kt
     - Récupère l'utilisateur retourné par createUserIfNotExists()
